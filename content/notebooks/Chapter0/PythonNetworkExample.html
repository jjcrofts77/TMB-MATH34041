0.4 Python Network Example#

In this section we run through a network analysis similar to what you are expected to do in your coursework, although this example is shorter and less developed than yours will be!

We shall consider a human connectome that has been constructed from MRI data using a parcellation (brain atlas) on 82 regions.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns # statistical data visualisation package
from sklearn.metrics.pairwise import pairwise_distances # machine learning package

# load C. elegans data
labels = pd.read_csv("https://raw.githubusercontent.com/jjcrofts77/TMB-MATH34041/main/content/Data/HumanBrainLabels.csv", header=None)
A = pd.read_csv("https://raw.githubusercontent.com/jjcrofts77/TMB-MATH34041/main/content/Data/HumanBrainAdjMat.csv", header=None)
positions = pd.read_csv("https://raw.githubusercontent.com/jjcrofts77/TMB-MATH34041/main/content/Data/HumanBrainPositions.csv", header=None)

# add labels
A.index = labels[0]
A.columns = labels[0]
positions.index = labels[0]

Let’s plot the brain network including spatial locations projected onto the \(x\) and \(y\) coordinates.

sns.set_style("whitegrid")
plt.figure(figsize=(20,12))

# plot edges
for i,row in enumerate(A.values):
  for j, columns in enumerate(row):
    if columns == 1:
      x = [positions.values[i,0],positions.values[j,0]]
      y = [positions.values[i,1],positions.values[j,1]]
      plt.plot(x,y,color="red",alpha=0.025)

# plot nodes and add labels
plt.plot(positions.values[:,0],positions.values[:,1],"o",alpha=0.4); #positions.values removes the labels

# degrees

Let’s compute the clustering coefficient for our network and then plot the network with node size proportional to the clustering.

import networkx as nx

# Compute the clustering coefficient and degree
G = nx.from_numpy_array(A.values)
C = nx.clustering(G)
D = nx.degree_centrality(G)
deg= np.array(list(D.items())) # convert from dict to numpy array
CC = np.array(list(C.items())) # convert from dict to numpy array

# set up 2 by 2 plot
fig = plt.figure(figsize=(20,12))
ax1 = fig.add_subplot(2,2,1) # deg plot
ax2 = fig.add_subplot(2,2,2) # clustering plot

# plot edges

    if columns == 1:
      x = [positions.values[i,0],positions.values[j,0]]

ax1.set_title("Degree",fontsize=20)

Is the human connectome a small-world network? To check this we need to compare the average path-length and clustering coefficient against a suitable ensemble of random networks, i.e. we say the network is small-world if it satisfies the following conditions

\[ \langle l\rangle \approx \langle l\rangle_\mathrm{rand} \quad \text{and} \quad \langle C\rangle >> \langle C\rangle_\mathrm{rand}. \]

Below we check whether these conditions are satisfied when the random networks are taken as Erdos-Reyni networks of the same size (i.e. \(n=82\) nodes).

# compute the average path-length and Watts-Strogatz clustering for the C. elegans network
import networkx as nx

G = nx.from_numpy_array(A.values)
n = G.number_of_nodes()
m = G.number_of_edges()

# Compute the clustering coefficient
C = nx.average_clustering(G)
l = nx.average_shortest_path_length(G)

# compute the average path-length and Watts-Strogatz clustering for 100 ER networks and compute their means
p = 2*m/(n*(n-1))
CC = 0
ll = 0
for i in range(100):
  ER = nx.erdos_renyi_graph(n,p)
  CC += nx.average_clustering(ER)
  ll += nx.average_shortest_path_length(ER)

print('The Watts-Strogatz CC for the network is ',C,'and for the comparable ER networks we have',CC/100)
print('The average path-length of the network is ',l,'and for the comparable ER networks we have',ll/100)

The Watts-Strogatz CC for the network is  0.6307218253654648 and for the comparable ER networks we have 0.32778470399066717
The average path-length of the network is  1.7208672086720866 and for the comparable ER networks we have 1.6723336344474546

We see that the network is small-world as expected.

It’s worth noting that we should probably take as our random ensemble a set of networks that preserve more than just the number of nodes (and the average number of edges). We could consider, for example, using the so-called configuration model which in addition to the above quantities, conserves the degree distribution. You can generate instances of the configuration model pretty easily within networkx.
